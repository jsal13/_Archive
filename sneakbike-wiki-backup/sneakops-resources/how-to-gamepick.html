TL;DR:

0. Consider the level of runner.
1. Pick a format.
2. Pick a console.
3. Pick some game(s), figure out good objectives.
4. Check with a Sneakops member to make sure it's good.
5. Iterate if necessary from the critiques of the previous step.
6. Bundle game + readme 

Step 0: What level race is it?

For beginners, we try to "be kind" and get them immersed in a game they may not have ever played before.  While an intermediate runner may easily sweep through the first four worlds of Super Mario Bros, it may not be the case for beginner runners.  Keep this in mind when picking games and objectives.

While there are no hard-and-fast rules, we have some general guidelines:

Beginner: Generally, we want to be nice, pick games which are perhaps less obscure / less nintendo-hard, and games which are "fun" over "frustrating".

Intermediate: Generally, we want to pick games which are a bit harder, a bit more obscure, and which have goals which are more challenging.  These individuals will have played a number of retro games and know the tropes; they'll be able to figure things out faster so there is less time spent on learning the controls and objectives compared to the beginner racers. 

Step 1: Pick a Format.

We have a few different formats at Sneakbike, each have varying levels of difficulty to set up, run, and pick for.

  • Single-Game Races: A single-game race consists of one game which should last around 40—60 minutes.  The idea here is to have the runners spend a lot of time immersing themselves in one particular game to get interested / learn the lore / master the controls / get pretty far.
    • Games which work well are: RPGs, Platformers.
    • Games which may work better in another format: Shoot'em'ups, Beat'em'ups, Puzzle Games.
  • Two-Game Race with Intermission: (The most common format.) Two game races should last around 50—70 minutes for gameplay (not including intermission).  The format is: the runners play one game, then intermission, then the players play the second game.  All runners start at the same time for both games.  This is a good format if you'd like one mid-size game and one short game, e.g. some puzzle game and some platformer.
  • Three-Game Relay: Three games are chosen and should last around 50—70 minutes.  There is no intermission, and the runners go from one game to the next as in a relay.  This is the most involved race for production and commentating, so use only when you have a good plan in mind and when the games will jive with one-another nicely.  

Step 2: Pick a Console.

While multi-console is possible, we typically opt to stick with one console at a time for ease of runner setup.  You can do multi-console if you've got a good idea or something, though!

There is no real restriction here, see what games we have and what is available to emulate on BizHawk.  For now, we try to avoid things like N64 (controller requirements) or PS1 (it's always great to emulate) and consoles after that.

Step 3: Pick some game(s), figure out an objective.

This is often the most difficult part: the objective should be short enough that the skill-level you're working with has a chance to complete it in the time allotted, but NOT short enough such that people are done in half that time.  Obviously, we are unable to see how much the runner knows, but here are some good general guidelines we've found to be helpful:

  1. Know thy level: play the game yourself and time yourself on the objective, then approximate how others of the relevant skill level would do.  You may want to say, “I'm an intermediate, so Beginners might take double the time on this game," for example.
  2. Ask someone else: get the help of a Sneakops (or someone not racing) and see how they do.  You may better approximate objectives from this.
  3. Make Good Objectives: pick an objective which is easy to understand and which a reasonable runner will know when it is complete while racing: beating stages, killing bosses, etc., are good examples of good objectives.

Step 4: Check with a Sneakops when you've got an idea.

Once you've got a plan, let us know and we'll see how reasonable it is.  Most of the time we'll iterate on your idea after trying it out.

Step 5: Iterate.

If necessary, a Sneakops might say something like, "This is fine, but if we have to extend it, what would a stretch goal be…?" or "What about if we did XYZ instead?"  Often, we'll think about “mercy kill” strategies where we can make the objective easier during the race if we begin to run long.  

We iterate on the ideas until an objective is agreed upon.

Step 6: README Bundling.

Once the objectives and games are agreed upon, fill out the important parts of the README Generator.

Additionally, please try to fill out some parts of the Game Info Tool for the commentators, to give them some quick links about the game to check out.  

Once you have these two HTML files, give them to the Sneakops and they'll distribute it to everyone who needs it, along with the ROM files.
